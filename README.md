# 3D Pose Estimation for Badminton Players

Estimates the real-world 3D position of badminton players' body joints from a single camera video. Uses known court geometry and ankle tracking to convert 2D pixel positions into accurate 3D coordinates in meters.

## Demo

---

## What It Does

Given a badminton match video with pre-tracked 2D keypoints and ankle world positions, this project:

1. Takes 17 body joint positions per player per frame (in pixels)
2. Uses the known ankle positions on the court (in meters) as a ground anchor
3. Estimates the full 3D pose of each player in real-world meters
4. Outputs an interactive 3D visualization you can view in any browser

---

## How It Works

The estimation uses a **geometric approach** — no AI model required:

- Ankles are placed at their known real-world court coordinates (derived from court line detection + homography)
- Each joint is assigned a height based on average human body proportions (e.g. nose = 1.70m, hip = 0.95m, knee = 0.50m)
- The horizontal offset of each joint from the ankle center is measured in pixels and scaled to meters
- Results are smoothed across frames to remove jitter

This approach works well for badminton because players are mostly upright and moving on a flat, well-defined court surface.

## Input Data Format

### `positions.json`
Generated by an upstream tracking pipeline. Contains per-frame data for each player:

```json
{
  "video_info": { "frame_count": 769, "fps": 30, "width": 1920, "height": 1080 },
  "court_info": { "width_meters": 6.1, "length_meters": 13.4 },
  "frame_data": {
    "1": {
      "player_0": {
        "keypoints_2d": [
          { "x": 450, "y": 320, "confidence": 0.92 },
          ...
        ],
        "ankles": [
          { "world_x": 4.2, "world_y": 9.8 },
          { "world_x": 3.9, "world_y": 9.6 }
        ]
      }
    }
  }
}
```

- `keypoints_2d` — 17 joints in COCO format (pixel positions + confidence score 0–1)
- `ankles` — real-world court coordinates in meters, derived from court homography

---

## Output Data Format

### `poses_3d_output.json`
```json
{
  "poses_3d": {
    "1": {
      "player_0": {
        "joints_3d": [[4.06, 8.82, 1.70], [4.05, 8.82, 1.68], ...],
        "joint_names": ["nose", "left_eye", "right_eye", ...]
      }
    }
  }
}
```

Each joint is `[X, Y, Z]` in meters. Origin is the top-left corner of the court.

## How to Run

All scripts are inside the `src/` folder. Run them from inside that folder:

```bash
cd src
```

### Step 1 — Run 3D Estimation

Reads `data/input/positions.json`, writes `data/output/poses_3d_output.json`.

```bash
python pose_3d_estimation.py
```

Expected output:
```
============================================================
3D POSE ESTIMATION — STANDARD GEOMETRIC METHOD
============================================================
✓ Total frames processed   : 719
✓ Successful estimations   : 1404
✓ Processing time          : 0.25s  (2874.0 fps)
============================================================
COMPLETE
============================================================
```

---

### Step 2 — Visualize (Choose One)

#### Option A — Interactive 3D in Browser (Recommended)
```bash
python viz2.py
```
Choose option `3` to save as HTML, or option `1` to open directly in browser.
Output HTML is saved to `visualizations/`.

#### Option B — Matplotlib 3D Viewer
```bash
python viz_3d.py
```
Opens a popup window. Can save as GIF.

#### Option C — Overlay on Original Video
```bash
python video_pose.py
```
Projects the 3D skeleton back onto the original video.
Output saved to `data/output/output_visualized.mp4`.

## Coordinate System

```
Origin (0,0) = Top-left corner of court (Player 1 side)

        0 ──────────── 6.1m
        |               |
        |   COURT       |        X = width  (0 → 6.1m)
        |               |        Y = length (0 → 13.4m)
        |               |        Z = height (0 = floor, up is positive)
       13.4m ──────────|
```

---

## Limitations

- Joint heights (Z) are fixed averages — jumping or deep crouching won't be reflected accurately
- Assumes a single static camera with no lens distortion
- Ankle tracking quality from the upstream pipeline directly affects 3D accuracy
- Works best when both players are fully visible
